# AI 学习提示词库

## 0. 避免重复造轮子的通用前置提示词

每次让 AI 写代码、设计方案、优化项目时，先加这一段：

```text
在给出实现前，请先检查 Python 标准库、当前框架内置能力、常见成熟第三方库、公司已有封装是否已经能解决，不要重复造轮子。
如果已有成熟方案，请优先使用成熟方案，并说明为什么。
如果为了学习需要手写最小实现，请明确标注“这是学习版，不建议生产直接使用”。
请同时给出：成熟方案、学习版最小实现、测试方法、常见坑。
```

---

## 1. 概念讲解

```text
请作为 Python Web 和爬虫老师解释「主题」。
要求：
1. 先讲它在工作中解决什么问题。
2. 再讲核心概念。
3. 给最小代码示例。
4. 给爬虫场景示例。
5. 给 Django 场景示例。
6. 最后出 3 道自测题。
```

## 2. 代码 Review

```text
请作为高级 Python 工程师 review 我的代码。
重点看：
1. bug
2. 异常处理
3. 日志
4. 可维护性
5. 测试缺失
6. 是否符合初级求职项目标准

请先列问题，再给修改建议，不要直接整段重写。
```

## 3. Django 项目阅读

```text
请帮我阅读这个 Django 项目。
请按以下顺序分析：
1. 启动入口
2. settings
3. 根路由
4. Middleware
5. View / ViewSet 基类
6. Serializer
7. Model / BaseModel
8. 统一响应
9. 异常处理
10. 异步任务
```

## 4. 爬虫请求分析

```text
这是浏览器请求和 Python 请求的差异。
请帮我分析为什么 Python 请求失败。
重点看：
1. URL
2. Method
3. Query
4. Body
5. Header
6. Cookie
7. Referer
8. 是否可能有加密参数
```

## 5. AI 应用设计

```text
请帮我设计一个 Django AI 接口。
功能：输入文章正文，返回摘要、关键词、分类和学习题。
要求：
1. 给接口设计。
2. 给数据模型。
3. 给 prompt 模板。
4. 给错误处理。
5. 给成本控制建议。
```
