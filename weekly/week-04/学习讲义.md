# Week 04 学习讲义：HTTP 与浏览器开发者工具

本周目标：看懂网页请求，知道数据从哪里来。

## 学习资料

- HTTP 概览：<https://developer.mozilla.org/en-US/docs/Web/HTTP/Overview>
- HTTP Methods：<https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods>
- HTTP Status：<https://developer.mozilla.org/en-US/docs/Web/HTTP/Status>
- HTTP Headers：<https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers>
- Chrome Network：<https://developer.chrome.com/docs/devtools/network/>

## 核心内容

HTTP 请求至少要看这些：

```text
URL
Method
Query String
Request Headers
Cookie
Request Payload
Response
Status Code
```

常见状态码：

```text
200 成功
301/302 跳转
403 禁止访问
404 不存在
429 访问太频繁
500 服务端错误
```

浏览器 Network 面板是爬虫的起点。优先找 `Fetch/XHR` 请求，因为很多网页数据来自接口 JSON。

## 每日安排

- Day 1：理解 URL、Query、Method。
- Day 2：理解 Header、Cookie、Referer、User-Agent。
- Day 3：理解状态码。
- Day 4：打开 Network 面板，看请求列表。
- Day 5：筛选 Fetch/XHR，查看 JSON 响应。
- Day 6：复制请求，分析差异。
- Day 7：完成 `http_debugger.py`。

## 练习题

1. 找一个网页，记录 3 个请求的 URL、Method、Status。
2. 找一个 XHR 请求，记录 Query 参数。
3. 对比浏览器请求和 Python 请求缺了哪些 Header。
4. 解释一次 403 或 429 可能的原因。

## 最终作业

创建：

```text
src/week04/http_debugger.py
```

功能：

- 输入 URL。
- 输出状态码。
- 输出响应头。
- 输出编码。
- 输出响应大小。
- 输出前 500 个字符。

## 验收标准

- 支持 timeout。
- 请求失败不崩溃。
- 能写一篇请求分析笔记。
