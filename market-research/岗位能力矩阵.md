# 岗位能力矩阵

## 1. 市场观察

当前国内相关岗位更常见的组合是：

```text
Python 后端 + Django/Flask/FastAPI + MySQL/Redis + Linux
Python 数据采集 + requests/Scrapy/Playwright + 数据清洗 + 反爬
AI 应用开发 + LLM API + RAG + Agent + Web 后端
```

单独“爬虫工程师”岗位仍有，但面更窄，且容易要求反爬、逆向、分布式采集。对你更稳的路线是：

```text
Python 后端/Django 为主，爬虫和 AI 应用作为加分项。
```

## 2. 能力分层

### 初级必须项

- Python 基础扎实。
- 会写 Django 接口。
- 会 MySQL 基础 CRUD。
- 会 Linux 基础命令。
- 会 Git。
- 会写简单爬虫。
- 会看日志和排查错误。

### 进阶加分项

- DRF。
- Celery / 定时任务。
- Redis。
- Scrapy。
- Playwright。
- Docker。
- pytest。
- API 文档。

### AI 加分项

- OpenAI 兼容 API。
- Prompt 设计。
- 结构化输出。
- RAG。
- 向量检索。
- AI 总结/分类/抽取。
- Django/FastAPI 接入 AI。

---

## 4. 学习优先级

| 优先级 | 内容 | 原因 |
|---|---|---|
| P0 | Python、HTTP、Django、MySQL | 求职基础盘 |
| P0 | requests、XPath、日志、异常 | 当前工作直接相关 |
| P1 | DRF、Middleware、公司框架阅读 | 提升项目阅读能力 |
| P1 | AI API、结构化输出、RAG | AI 应用岗位加分 |
| P2 | Scrapy、Playwright、Celery | 项目深度 |
| P2 | Docker、部署、CI | 面试和项目完整性 |

---

## 5. 投递方向建议

优先投：

```text
Python 开发工程师
Python 后端开发工程师
Django 开发工程师
数据采集工程师
数据处理工程师
AI 应用开发工程师（初级）
RAG 应用开发工程师（初级）
```

谨慎投：

```text
高级爬虫逆向工程师
算法工程师
大模型训练工程师
NLP 算法工程师
```

原因：这些岗位通常要求更强的逆向、算法、深度学习或分布式经验，不是当前最短路径。
