# 实战反爬与市场路线

这份文档用于补足原学习路线中过于学院化的问题。真实市场中的 Python 爬虫、数据采集、Django 数据平台岗位，通常不是只要求会 `requests + XPath + 入库`，而是要求你能处理动态页面、验证码、加密参数、账号态、风控、稳定性和合规。

重要边界：

```text
本路线只用于合法授权的数据采集、公司内部业务系统、公开数据处理、测试环境验证和安全合规研究。
不要绕过登录授权、付费墙、隐私保护、服务条款、强风控系统或法律禁止的访问控制。
```

---

## 1. 当前市场更贴近真实工作的能力模型

从最近的岗位描述看，爬虫/数据采集岗位常见要求已经变成：

```text
Python 基础
HTTP / 浏览器请求分析
requests / httpx
Scrapy / Scrapy-Redis
Playwright / Selenium
XPath / CSS Selector / 正则
JS 逆向基础
加密参数分析
验证码处理
代理与限速
账号态与 Cookie 管理
MySQL / Redis / Kafka
任务调度与监控
Django / Flask / FastAPI 数据平台
Docker / Linux 部署
合规意识
```

你的学习路线应该从“学科顺序”改成“问题场景顺序”：

```text
遇到一个网站 -> 分析请求 -> 判断数据来源 -> 判断反爬类型 -> 选择采集方案 -> 稳定落库 -> 做成接口/后台 -> 监控与维护
```

---

## 2. 真实工作中的 8 类采集场景

## 场景 1：静态 HTML 页面

特征：

- 源码里直接有数据。
- 不需要登录。
- 不需要执行 JavaScript。

优先方案：

```text
requests/httpx + BeautifulSoup/lxml + JSONL/数据库
```

训练重点：

- HTTP 基础
- HTML 解析
- 字段清洗
- 失败重试
- 去重入库

---

## 场景 2：接口 JSON 页面

特征：

- 页面数据来自 XHR/fetch 接口。
- Network 面板能看到 JSON。
- 参数通常包括分页、关键词、排序。

优先方案：

```text
浏览器 Network 分析 -> 还原接口请求 -> requests/httpx 调用
```

训练重点：

- Network 面板
- Header / Cookie / Referer
- 分页参数
- 请求重放
- 接口返回结构分析

---

## 场景 3：动态渲染页面

特征：

- HTML 源码没有数据。
- 页面由 React/Vue/Angular 等前端渲染。
- 需要滚动、点击、等待加载。

优先方案：

```text
先找接口；找不到接口再用 Playwright
```

训练重点：

- Playwright
- 元素等待
- 滚动加载
- 网络监听
- 截图排障

---

## 场景 4：验证码

特征：

- 登录、搜索、翻页或高频访问时出现验证码。
- 可能是文字验证码、滑块验证码、点选验证码。

学习工具：

- ddddocr：<https://pypi.org/project/ddddocr/>

ddddocr 常见用途：

```text
文字验证码识别
滑块缺口识别
目标检测类验证码的辅助识别
```

学习要求：

- 只在授权测试、内部系统、学习样例中使用。
- 不把验证码识别当作万能方案。
- 先降低访问频率、增加缓存、减少重复请求。
- 如果网站明确禁止自动化访问，不继续绕过。

练习项目：

```text
projects/captcha_lab/
```

功能：

- 准备本地验证码图片样例。
- 使用 ddddocr 做文字识别。
- 记录识别结果、耗时、准确率。
- 提供一个 FastAPI/Django 内部 OCR 服务接口。

验收标准：

- 能处理本地样例。
- 有准确率统计。
- 有失败样例归档。
- 不连接真实未授权网站做绕过。

---

## 场景 5：JS 加密参数

特征：

- 接口参数里有 `sign`、`token`、`timestamp`、`nonce`、`signature`。
- 请求参数每次变化。
- 直接复制请求很快失效。

学习重点：

```text
JavaScript 基础
浏览器断点调试
搜索参数名
调用栈分析
常见编码与摘要算法
Node.js 执行 JS
Python 调用 JS
```

必须掌握的基础概念：

- Base64
- URL encode/decode
- MD5/SHA 系列摘要
- AES/RSA/DES 的概念区别
- 时间戳和随机数
- HMAC
- Webpack 打包代码阅读

练习项目：

```text
projects/js_signature_lab/
```

功能：

- 自己写一个本地 Django/FastAPI 测试接口。
- 前端用 JS 生成签名。
- Python 端尝试复现签名。
- 记录分析过程。

验收标准：

- 能解释签名由哪些字段组成。
- 能用 Python 或 Node.js 复现测试签名。
- 不针对真实网站发布绕过细节。

---

## 场景 6：账号态、Cookie、Session

特征：

- 需要登录后才能看到数据。
- Cookie 过期。
- 请求频繁后账号异常。

学习重点：

- Cookie / Session
- CSRF Token
- 登录态保存
- 多账号风险
- 访问频率控制
- 合规审批

实践要求：

```text
优先使用正式 API、公司授权账号、内部系统或测试环境。
真实业务系统必须遵守公司数据权限和审计要求。
```

---

## 场景 7：反爬风控与稳定性

特征：

- 403、429、验证码、跳转、空数据、封 IP。
- 同样请求一会儿成功一会儿失败。

学习重点：

- 限速
- 缓存
- 失败重试
- 指数退避
- 代理质量评估
- 请求指纹
- 任务监控
- 告警

工程化要求：

```text
不能只写能跑一次的脚本，要写能长期维护的采集任务。
```

---

## 场景 8：数据平台化

特征：

- 采集只是第一步。
- 业务真正需要的是查询、审核、导出、监控、任务管理。

优先方案：

```text
Scrapy/Playwright 采集 -> MySQL/PostgreSQL -> Redis 队列/去重 -> Django/DRF 后台和接口 -> Docker 部署
```

训练重点：

- Django Admin
- DRF API
- 任务表
- 失败记录表
- 采集配置表
- 数据质量报告

## 5. 面试需要能讲清楚的问题

爬虫基础：

- requests 和 Playwright 分别适合什么场景？
- 如何判断数据来自 HTML 还是接口？
- 如何做翻页采集？
- 如何避免重复采集？

反爬与稳定性：

- 遇到 403/429 怎么排查？
- 验证码出现时先考虑哪些非对抗方案？
- 什么情况下会用 OCR？
- 如何处理 Cookie 过期？
- 如何设计失败重试？

JS 加密：

- 什么是签名参数？
- timestamp、nonce、sign 常见作用是什么？
- 如何用 DevTools 找到签名生成位置？
- Python 如何复现 MD5/HMAC 类签名？

工程化：

- 如何设计采集任务表？
- 如何记录失败 URL？
- 如何做任务监控？
- 为什么需要 Redis？
- Django Admin 在数据平台里有什么价值？

合规：

- 什么数据不能采？
- 如何控制访问频率？
- 公司内部数据权限如何处理？
- 为什么不能随便绕过登录和验证码？

## 6. 学习优先级

如果时间有限，优先级如下：

```text
P0：Python 工程基础、HTTP、Network、requests、数据清洗、Django/DRF
P1：Scrapy、Playwright、数据库、Redis、任务调度
P2：验证码 OCR、JS 签名、浏览器断点、Node.js 辅助执行
P3：分布式采集、Kafka、复杂风控、APP 抓包、深度逆向
```

你当前最应该补的是：

```text
1. Network 请求分析能力
2. Scrapy/Playwright 的组合使用
3. Django 数据平台能力
4. ddddocr 这类验证码识别工具的安全使用
5. JS 加密参数的基础分析方法
```

这样学习路线会更贴近实际市场，也更容易在简历和面试中讲出真实项目价值。
