# 学习总计划

## 1. 学习目标

你的目标应从“会写爬虫脚本”升级为：

```text
能胜任 Python 后端 / Django / 数据采集 / AI 应用开发相关岗位。
```

真正开始学习时，请优先使用最外层的 `详细学习资源与周计划.md`。它已经把每周要看的官方文档、重点章节、作业和验收标准拆开；本文件负责说明总方向，`weekly/20周学习计划.md` 负责做快速索引。

具体能力包括：

1. Python 工程基础：函数、模块、异常、日志、测试、项目结构。
2. 数据采集能力：HTTP、requests、解析、反爬意识、数据清洗、入库。
3. Django 工作能力：URL、View、Model、ORM、Middleware、DRF、公司魔改框架阅读。
4. 数据能力：SQLite、MySQL、Redis、任务状态、去重、分页、查询。
5. AI 使用能力：提示词、AI code review、低成本学习闭环。
6. AI 开发能力：API 调用、结构化输出、RAG、简单 Agent、Django AI 接口。
7. 求职能力：项目包装、简历表达、面试题、岗位匹配。

---

## 2. 职业定位

建议优先定位三类岗位：

### A. Python 后端开发

关键词：

```text
Python、Django、DRF、MySQL、Redis、Linux、接口开发、后台系统
```

适合原因：Django 是你公司项目的主要技术栈，学习收益最大。

### B. Python 数据采集 / 数据工程初级

关键词：

```text
爬虫、requests、Scrapy、Playwright、数据清洗、反爬、MySQL、定时任务
```

适合原因：贴近你当前工作，但不要只投“纯爬虫”，岗位面会窄。

### C. AI 应用开发初级

关键词：

```text
LLM、Prompt、RAG、Agent、OpenAI兼容API、Django/FastAPI、知识库问答
```

适合原因：AI 是趋势，但需要和 Python/Web 项目结合，不能只会聊天。

## 3. 学习周期

建议 20 周。

```text
第 1-3 周：Python 工程基础
第 4-7 周：HTTP 与爬虫核心
第 8-11 周：Django 与公司魔改框架
第 12-13 周：AI 使用能力与低成本学习法
第 14-17 周：AI 开发基础
第 18-20 周：综合项目与求职准备
```

每周最低交付：

```text
1 篇学习笔记
1 个可运行练习
1 个作业
1 次复盘
1 次 Git commit
```

---

## 4. 每天学习要求

每天 1.5 到 2 小时：

```text
20 分钟：看讲义 / 官方文档
40 分钟：写代码
20 分钟：调试和记录问题
20 分钟：做题或复盘
10 分钟：提交 Git 或整理 README
```

如果当天很忙，最低要求：

```text
写 20 行代码 + 记录 3 个问题
```

---

## 5. 每周复盘要求

每周必须回答：

1. 本周我能独立写出什么？
2. 哪个知识点我只是看懂了，但写不出来？
3. 哪个错误出现超过 2 次？
4. 这个知识点在爬虫里怎么用？
5. 这个知识点在 Django 项目里怎么用？
6. 这个知识点和 AI 应用有什么关系？
7. 本周是否完成 Git 提交？

---

## 6. Codex 到期后的学习方式

不要依赖单一 AI 工具。

建议组合：

```text
本地 Markdown 文档：主教材
官方文档：权威资料
pytest：自动检查作业
普通 AI：解释和出题
高级 AI：每周一次 code review / 架构 review
GitHub：沉淀作品
```

你要逐渐形成能力：

```text
AI 帮我学，但不替我学。
AI 帮我写，但我能判断对错。
AI 帮我 review，但我自己能修。
```
