# Python Web 与爬虫开发系统学习计划

适用对象：已经接触 Python 或正在用 Python 做爬虫，但希望系统补足基础、Web、爬虫、工程化能力的人。

学习方式：每个阶段都包含“学什么、练什么、交什么作业、如何复盘”。不要只看教程，必须写代码、调试、整理笔记。

建议周期：16 周。每天 1.5 到 2 小时，每周 5 天。时间紧可以拉长到 24 周。

---

## 总目标

学完后你应该具备这些能力：

1. 能独立写出结构清晰、可维护的 Python 程序。
2. 能读懂常见 Python 项目结构，理解模块、包、异常、日志、配置、测试。
3. 能使用 requests、httpx、BeautifulSoup、lxml、XPath、正则、Playwright、Scrapy 完成不同类型的爬虫。
4. 能理解 HTTP、Cookie、Session、Header、代理、编码、压缩、重定向、异步请求。
5. 能开发基础 Web API，理解路由、请求响应、数据库、认证、中间件、后台任务。
6. 能把爬虫和 Web 结合，做一个“采集、清洗、入库、查询、展示”的完整项目。
7. 能进行基本部署、日志排查、任务调度和性能优化。

---

## 学习原则

1. 每天必须写代码。只看视频或文章不算学习完成。
2. 每周必须有一个可运行的小项目或脚本。
3. 学完一节后，先关掉资料，凭记忆重写核心代码。
4. 每周最后一天只做复盘：整理错题、重构代码、补测试。
5. 每个阶段都保留一个 `README.md`，记录你学到了什么、踩了什么坑。

建议目录结构：

```text
python-learning/

```

# 最后给你的学习建议

你的主线是爬虫，所以不要把 Python 学成纯语法课。每学一个基础点，都问自己：它在爬虫或 Web 中怎么用？

- 字典：保存字段、请求头、配置。
- 函数：封装请求、解析、保存。
- 类：表示任务、爬虫、数据模型。
- 异常：处理请求失败、解析失败、入库失败。
- 日志：定位线上采集问题。
- 数据库：保存采集结果和任务状态。
- Web：把数据和任务能力暴露出来。
- 异步：提升大量 I/O 请求效率。

真正的能力不是“我看过”，而是“我能在没有教程的情况下，把一个问题拆开、写出来、调通、整理好”。这份计划就按这个目标走。
